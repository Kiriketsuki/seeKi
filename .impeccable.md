## Design Context

### Users

Non-technical staff who need to read production database contents without learning SQL or installing DBeaver. Primary origin case: operations people at My Database browsing 20+ tables and 500K+ rows to answer business questions. They sit at a desk in daylight, open SeeKi, point at a table, and expect a spreadsheet. They do not want a query editor, a schema tree with joins, or anything that looks like a developer tool.

**Job to be done**: find the row, read the value, trust that what is shown is what is in the database.

### Brand Personality

Three words: **simple, trustworthy, source-of-truth.**

Voice: quiet confidence. The product never shows off. It never says "powerful" or "advanced." It behaves like a well-kept ledger — the data is the hero, the chrome gets out of the way.

Emotional goal on first grid load: *"oh, that's easy."* Relief and comprehension, not excitement. The user should feel like they already know how to use it.

### Aesthetic Direction

**Theme**: light and dark. Light is the default (office daylight context); dark is a first-class alternative, not an afterthought.

**Visual tone**: warm-neutral, paper-like, archival. Cream surfaces over cold gray. Ink-navy text instead of pure black. Accent color used sparingly for state, not decoration.

**Glassmorphism**: retained but only where it earns its place — translucent chrome (sidebar, toolbar, statusbar) floating over the data layer to reinforce that the grid is the primary surface. Do NOT apply glass to interior cards, modals, dropdowns, or anywhere decorative. If blur is not conveying "chrome over data," remove it.

**Signature details (keep)**:
- Hatched NULL cells — the single most distinctive visual cue, do not replace
- Yes/No badges for booleans — keep the semantic green/red, refine if drifting toward generic
- Zebra striping — subtle, warm-tinted, never gray

**Anti-references** (must NOT feel like any of these):
- DBeaver — dense, toolbar-heavy, engineer-first
- MySQL Workbench — institutional, cold, tree-panel-dominant
- phpMyAdmin — 2008-web-app, tabbed, utilitarian
- Airtable / Notion databases — over-branded, colorful, collaborative-first (SeeKi is read-only and local)

**Typography direction**: retain a data-entry / tabular feel — monospace or mechanical sans in the grid so numeric columns align naturally — but replace the current Inter + JetBrains Mono pairing. Both are reflex defaults.

Candidates to explore (not final, but direction-setting):
- UI: **Hanken Grotesk** (humanist, warm, open-license) or **Söhne** (paid) — both avoid the Inter monoculture while keeping neutrality
- Tabular/mono: **Monaspace Neon** (GitHub, free, variable axes for subtle character) or **Commit Mono** (humanist mono, free) — both read as "data" without the JetBrains tell

Final choice to be locked during first typography pass. Neither font from the reflex-reject list is acceptable.

**Color**:
- Current palette (`#F5F0EB` cream / `#2F4858` ink-navy / `#ff9500` orange) stays in spirit but will be re-expressed in OKLCH, with neutrals tinted toward the ink-navy hue for subconscious cohesion.
- Orange accent used for state (active sort, selection, live status) — never for decoration or gradients. Aim for <10% of visual weight.
- Dark theme is a genuine counterpart: warm charcoal surface (not pure black), softened ink text, same accent hue but chroma-reduced for the darker background.

### Design Principles

1. **The grid is the hero.** Every other surface recedes. Chrome is translucent and quiet; grid cells are opaque and authoritative.
2. **Read before write — in fact, never write.** SeeKi is read-only. Visual language avoids anything that implies mutation (no pencil icons, no primary-action buttons competing for attention, no "save" affordances).
3. **No SQL vocabulary visible.** Sort, search, filter — yes. "Query," "WHERE," "JOIN," "schema" — no. The language is spreadsheet, not database.
4. **Tasteful glass, not decorative glass.** Blur earns its place only when separating chrome from the data layer. Interior surfaces are solid.
5. **Warm over cold.** Cream over gray, ink-navy over pure black, softened accents. The product should feel like paper and daylight, not aluminum and neon.
6. **Accessibility floor: WCAG AA.** All text and interactive state meets AA contrast in both themes. Respect `prefers-reduced-motion`. No information conveyed by color alone (Yes/No badges carry a shape/label, not just hue).
